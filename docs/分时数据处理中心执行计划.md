## 分时数据处理中心执行计划（执行版）

### 1. 启动阶段（T0 ~ T0+1周）
1. **Kickoff 会议**
   - 参与：产品、数据、策略、后端、前端、测试、运维。
   - 议程：确认重构愿景、优先级、时间表、资源配置、风险点。
   - 输出：会议纪要、决策记录、行动项。
2. **角色与责任矩阵 (RACI)**
   - 明确每个模块负责人、备份人、协作接口。
   - 建立跨团队沟通渠道（周会/日报/看板）。
3. **基线评估**
   - 现状审计：数据源稳定性、延迟、缺失率、算法准确率、服务 QPS。
   - 技术债盘点：列出必须修复的遗留问题，避免迁移后放大风险。
4. **流程治理**
   - 建立 ADR 模板、文档模板、代码评审规范、CI 要求。
   - 设立集中项目看板（如 Jira/语雀/飞书）跟踪需求与任务。

### 2. 阶段路线图
| 阶段 | 时间范围 | 目标 | 里程碑/验收 |
| --- | --- | --- | --- |
| P0 数据骨架 | T0+1w ~ T0+4w | 完成多源采集、流式缓冲、清洗闭环 | Stream 数据持续 24h 稳定运行；Prometheus 指标上线；历史落地成功 |
| P1 机会引擎 MVP | T0+4w ~ T0+8w | 打通三条核心策略的信号链路 | 策略信号实时推送；机会列表对外服务；策略准确性初步评估 |
| P2 回测评估 | T0+8w ~ T0+12w | 搭建回测沙箱与报表体系 | 5 套策略完成回测；报表展示上线 |
| P3 前端重构 | T0+10w ~ T0+14w | 完成新驾驶舱与策略管理界面 | 新 UI 切换成功；旧版逐步下线 |
| P4 扩展演进 | T0+14w+ | 增强风控、策略商店、模型化 | 风控预警上线；策略版本管理稳定运行 |

### 3. Kickoff 前置准备
- 整理现有数据源接口文档、限流策略、账号权限。
- 梳理现有策略逻辑、指标计算方法、历史验证结果。
- 收集运维环境信息：服务器配置、部署方式、监控现状、告警通道。
- 确认可复用的基础设施（Redis、Kafka、ClickHouse 等）。

### 4. 每周例会建议
- **周例会**：回顾进度、风险、需求变更；展示关键指标；决定下周重点。
- **技术专题会**：针对数据管道、策略框架、前端架构等专题进行深入讨论。
- **评审会议**：每个阶段结束进行阶段复盘与下一阶段立项评审。

### 5. 沟通与文档
- 使用统一模板输出：
  - 会议纪要（Meeting Notes）
  - ADR（Architecture Decision Record）
  - 技术方案（Tech Spec）
  - 测试报告（QA Report）
  - 部署说明（Deployment Guide）
- 文档统一存放于 `docs/` 并同步到团队知识库。

### 6. KPI 与监控指标 (初版)
- **数据层**：抓取成功率、延迟、中断次数、缺失率、异常值比例。
- **策略层**：信号数量、平均置信度、命中率、错误提醒次数、信号滞后。
- **服务层**：API QPS、响应时间、错误率、推送成功率。
- **体验层**：前端加载时长、交互延迟、用户留存/使用时长（若有统计）。

### 7. 风险管理
- 建立风险登记表，记录风险描述、概率、影响、应对方案、责任人。
- 每周更新风险状态，确保重大风险上报并跟进。
