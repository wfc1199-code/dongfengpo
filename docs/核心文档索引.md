## 🔄 文档使用场景

### 场景 1: 开始新功能开发

**查阅顺序**：

1. PRD → 确认功能需求和验收标准
2. 技术设计文档 → 了解架构和实现方案
3. 实施计划 → 查看具体实施步骤
4. 开发流程规范 → 遵守开发流程
5. Git 工作流指南 → 创建分支、提交代码

### 场景 2: 代码审查

**查阅内容**：

1. 开发流程规范 → 代码审查 Checklist
2. PR 模板 → 检查 PR 是否完整
3. PRD → 验证功能是否符合需求
4. 技术设计 → 验证实现是否符合架构

### 场景 3: Bug 修复

**查阅内容**：

1. PRD → 确认预期行为
2. 技术设计 → 理解系统架构
3. Git 工作流 → 创建 bugfix 分支
4. 开发流程规范 → 遵守测试要求

### 场景 4: 技术方案设计

**查阅内容**：

1. PRD → 理解业务需求
2. 现有技术设计 → 保持架构一致性
3. 开发流程规范 → 了解质量标准
